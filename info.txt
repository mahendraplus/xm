Homepage URL  : https://mahendraplus.github.io/xm/
API Base URL  : https://mahendraplus-api-db-in.hf.space
GitHub Repo   : mahendraplus/xm

Description:
  Go-Biz is a credit-based mobile number lookup and KYC data retrieval
  platform. Users register, purchase credits via Razorpay, and then search
  for personal information (name, father's name, address, email, alternate
  number, DOB, gender, carrier, circle, Aadhaar-linked ID) associated with
  Indian mobile numbers. The platform provides a REST API for programmatic
  access alongside a rich web UI.

6. AUTHENTICATION & AUTHORIZATION
================================================================================

Method      : JWT Bearer Token (HS256)
Flow        :
  1. User registers via POST /api/auth/register
  2. Account is auto-activated (configurable by admin)
  3. User logs in via POST /api/auth/login → receives JWT token
  4. Token stored in Zustand authStore (persisted to localStorage)
  5. Every API request attaches "Authorization: Bearer <token>" via Axios interceptor
  6. On app mount, profile is re-hydrated from GET /api/user/profile
  7. If token is invalid/expired, user is automatically logged out

Roles       : "user" (standard), "admin" (full access)
Security    : Max login attempts (5), block duration (10 minutes), configurable
Forgot Pass : Users can request password reset; admin resolves via admin panel

  GET  /api/pricing                                  — Pricing tiers (basic: ₹1, address: ₹3.5, full_kyc: ₹10, deep: ₹25)

--- User Endpoints (Requires User Token) ---
  GET  /api/user/profile                             — User profile (name, email, role, credits, searches, API key, status)
  POST /api/user/search                              — Search by mobile number with field selection
  GET  /api/user/history                             — Search history with billing details

--- Payment Endpoints (Requires User Token) ---
  POST /api/payments/create-order                    — Create Razorpay payment order
  GET  /api/payments/history                         — Payment transaction history

--- Chat Endpoints ---
  POST /api/chat/send                                — User sends a support message
  GET  /api/chat/history                             — User gets chat history

--- Admin Endpoints (Requires Admin Token) ---
  GET  /api/admin/debug-token-tmp                    — Get temporary admin token (internal)
  POST /api/admin/users/{email}/activate             — Activate a user account
  POST /api/admin/users/{email}/credits/add          — Add credits to user wallet
  GET  /api/admin/stats                              — System dashboard (users, searches, pending items, health)
  GET  /api/admin/settings                           — System settings (auto-activate, maintenance mode, limits, Razorpay mode)
  GET  /api/admin/config/prices                      — Per-field pricing configuration
  GET  /api/admin/logs/requests                      — Recent HTTP request logs (method, path, status, duration, IP)
  GET  /api/admin/logs/system                        — System info (neofetch output, container env vars)
  GET  /api/chat/admin/list                          — List all user chats (with unread indicators)
  GET  /api/chat/admin/history/{email}               — View a specific user's chat history
  POST /api/chat/admin/send/{email}                  — Admin sends reply to a user

8. SEARCH FEATURE — FIELD PRICING
================================================================================

Each search costs a base fee + per-field charges:

  Field       | Price (₹/credits)
  ------------|-------------------
  base_fee    | 1.0
  name        | 5.0
  fname       | 10.0
  address     | 25.0
  email       | 15.0
  alt         | 20.0
  dob         | 5.0
  gender      | 2.0
  carrier     | 1.0
  circle      | 1.0
  id (Aadhaar)| 100.0
  ------------|-------------------
  ALL fields  | 177.0 (typical)

Users can select individual fields or "ALL" to control cost.
Returned data includes: mobile, name, fname, address, email, alt, dob,
gender, carrier, circle, id (Aadhaar-linked 12-digit number).

9. PAYMENT INTEGRATION
================================================================================

Gateway     : Razorpay (Indian payment gateway)
Modes       : Test mode / Live mode (configurable by admin)
Currency    : INR (Indian Rupees)
Limits      : Min deposit ₹100, Max deposit ₹1,00,000
Flow        :
  1. User initiates payment → POST /api/payments/create-order
  2. Razorpay order created with amount in paise (₹500 = 50000 paise)
  3. Frontend opens Razorpay checkout modal
  4. On success, credits are added to user wallet
  5. On modal dismiss, order remains in "created" status
  6. Payment history tracked and visible in Dashboard
